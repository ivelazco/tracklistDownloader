    **Mapping rules (concrete):**
    - If `err` has numeric `statusCode === 401`: message must include substring **`Spotify credentials`** and **`config/local.json`** and suggest checking **`clientId`** and **`clientSecret`**.
    - If `statusCode === 403`: message must include **`permission`** OR **`forbidden`** AND **`public`** (indicating playlist may be private or inaccessible with client credentials).
    - If `statusCode === 404`: message must include **`Playlist ID:`** followed by the actual `ctx.playlistId` when provided (keep existing ID-in-message behavior).
    - If `statusCode === 429`: message must include **`rate limit`**; if `err.headers` is an object and `retry-after` / `Retry-After` header exists, append the retry seconds value to the message (read header case-insensitively).
    - If `statusCode` is another number: include literal substring **`Spotify API`** and the numeric code (e.g. `status 500`).
    - For `operation === 'token'` (thrown from `clientCredentialsGrant`): if no `statusCode`, message must include **`clientId`** and **`clientSecret`** and **`config/local.json`** (invalid client / bad secret).

    Use type guards: `typeof err === 'object' && err !== null && 'statusCode' in err` for API errors; safely read `headers` if present.
